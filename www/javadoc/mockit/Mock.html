Annotation Type Mock


  • @Inherited
@Retention(value=RUNTIME)
@Target(value=METHOD)
public @interface Mock
    Used inside a mock class to indicate a mock method whose implementation will temporarily replace the implementation of a matching "real" method.

    The mock method must have the same name and the same parameters as the matching real method, except for an optional first parameter of type Invocation; if this extra parameter is present, the remaining ones must match the parameters in the real method. The mock method must also have the same return type as the matching real method.

    Method modifiers (including public, final, and even static), however, don't have to be the same. Checked exceptions in the throws clause (if any) can also differ between the two matching methods. A mock method can also target a constructor, in which case the previous considerations still apply, except for the name of the mock method which must be "$init".

    A mock method can specify constraints on the number of invocations it should receive while in effect (ie, from the time a real method/constructor is mocked to the time it is restored to its original definition).

    The special mock methods void $init(...) and void $clinit() correspond to constructors and to static class initializers, respectively. (Notice that it makes no difference if the real class contains more than one static initialization block, because the compiler merges the sequence of static blocks into a single internal "<clinit>" static method in the class file.) Mock methods named $init will apply to the corresponding constructor in the real class, by matching the declared parameters; just like regular mock methods, they can also have a first parameter of type Invocation.

    • Element Detail

      • invocations

        public abstract int invocations
        Number of expected invocations of the mock method. If 0 (zero), no invocations will be expected. A negative value (the default) means there is no expectation on the number of invocations; that is, the mock can be called any number of times or not at all during any test which uses it.

        A non-negative value is equivalent to setting minInvocations and maxInvocations to that same value.

        Default:
        -1

      • reentrant

        @Deprecated
public abstract boolean reentrant
        Deprecated. This attribute will be removed in a future release. Existing mock methods marked as reentrant should instead have an Invocation parameter. Uses of the "it" field should be replaced with calls to corresponding methods in the Invocation object, in particular Invocation.getInvokedInstance() to obtain the "it" instance, and Invocation#proceed(...) to execute the real method.
        Indicates whether or not the mock method implementation is allowed to call the corresponding real method on the it field before it returns from an execution. (The it field is an instance field defined in the mock class with name "it" and of the same type as the mocked class, which will be set to the real class instance for each indirect call to a mock method.) By default, such reentrant calls are not allowed because they lead to infinite recursion, with the mock method calling itself indirectly through the mocked method.

        When allowed to make such calls, the mock method effectively behaves as advice to the corresponding real method.

        As an alternative, consider adding a first parameter of type Invocation to the mock method, making it automatically reentrant.

        Default:
        false
