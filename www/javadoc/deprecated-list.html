Deprecated API

Contents

  • Deprecated Annotation Types
  • Deprecated Methods
  • Deprecated Annotation Type Elements
  • Deprecated Annotation Types 
    Annotation Type and Description
    mockit.MockClass
    This annotation will be removed in a future release. Existing mock classes should extend the MockUp<T> base class instead of using the annotation. In cases where the MockClass.stubs() attribute is specified, the methods/constructors to be stubbed out should instead have corresponding @Mock methods with empty bodies.
  • Deprecated Methods 
    Method and Description
    mockit.Mockit.setUpMock(Class, Class)
    Use MockUp.MockUp() or MockUp.MockUp(Class) instead.
    mockit.Mockit.setUpMock(Class, Object)
    Use MockUp.MockUp() or MockUp.MockUp(Class) instead.
    mockit.Mockit.setUpMock(Object)
    Use MockUp.MockUp() and MockUp.getMockInstance() (when mocking an interface) instead.
    mockit.Mockit.setUpMocks(Object...)
    Use MockUp instead.
    mockit.Mockit.stubOut(Class...)
    Use the UsingMocksAndStubs annotation instead.
    mockit.Mockit.stubOutClass(Class, boolean, String...)
    Use MockUp instead, with empty Mock methods for the methods and constructors to be stubbed out.
    mockit.Mockit.stubOutClass(Class, String...)
    Use MockUp instead, with empty Mock methods for the methods and constructors to be stubbed out.
  • Deprecated Annotation Type Elements 
    Annotation Type Element and Description
    mockit.Capturing.classNames
    This attribute will be removed in a future release.
    mockit.Capturing.inverse
    This attribute will be removed in a future release.
    mockit.Mocked.realClassName
    This attribute will be removed in a future release. For classes that extend/implement an accessible base type, the Capturing annotation should be used instead.
    mockit.Mock.reentrant
    This attribute will be removed in a future release. Existing mock methods marked as reentrant should instead have an Invocation parameter. Uses of the "it" field should be replaced with calls to corresponding methods in the Invocation object, in particular Invocation.getInvokedInstance() to obtain the "it" instance, and Invocation#proceed(...) to execute the real method.
